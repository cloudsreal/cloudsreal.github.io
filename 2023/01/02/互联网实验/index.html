VLAN

VLAN（Virtual Local Area Network）即虚拟局域网，是将一个物理的LAN在逻辑上划分成多个广播域的通信技术。VLAN内的主机间可以直接通信，而VLAN间不能直接通信，从而将广播报文限制在一个VLAN内。

ACL

参考09：ACL实验

访问控制列表ACL（Access Control List）是由一条或多条规则组成的集合。所谓规则，是指描述报文匹配条件的判断语句，这些条件可以是报文的源地址、目的地址、端口号等。 ACL本质上是一种报文过滤器，规则是过滤器的滤芯。设备基于这些规则进行报文匹配，可以过滤出特定的报文，并根据应用ACL的业务模块的处理策略来允许或阻止该报文通过。

问题整理

NAT

NAT的作用

较彻底解决IPv4地址空间不足的核心机制，将内部（本地）网络地址转换为注册的网络地址。

NAT的三种方式

NAT的实现方式有三种，即静态转换Static Nat、动态转换Dynamic Nat和端口多路复用OverLoad。

  • 静态NAT（static NAT）：固定的内部地址(internal address)到注册地址(registered address)的映射(一开始就写死)
  • 动态NAT（Dynamic NAT）：映射以先到先得的方式动态进行(不是写死，配一个地址池)
  • 端口地址转换PAT（Port Address Translater）：端口地址转换用于允许许多内部用户共享一个“内部全局”地址(基于Socket映射，而不是IP地址，多个内网主机映射到一个公网地址)

VLAN

引入vlan的原因

  • 划分小的广播域，对接入设备进行逻辑上的分组，使PC0、PC2（vlan10）可以直接通过交换机通信，PC1、PC3（vlan20）直接通信，不同vlan间不能直接通信，需要通过路由器转发，便于管理
  • 非同vlan用户不能直接通信，增强安全性
  • 解决广播风暴问题，提高网络性能（其实不太懂）

简述实验中的vlan

  • vlan划分如1所述，路由器Router1的F1/0划分出两个子接口F1/0.10和F1/0.20，分别作为两个网段（192.168.10.0和192.168.20.0）的网关
  • 路由器和交换机之间通过trunk技术连接，节省路由器端口
  • 交换机之间也通过trunk连接，因为需要传输不同vlan的帧（不是报文）

其他注意事项

  1. 帧封装格式（encapsulation）：IEEE 801.1q（对帧进行打标记的方式，在第二层通过交换机进行）

  2. vlan间通信需要使用ARQ协议查询网关地址（也不太懂，来个佬浇浇）

  3. 配置vlan时用的不是subnet mask（子网掩码），而是wild mask（检查不匹配的位）

  4. vlan对PC是透明的，即vlan只需要在交换机和路由器上配置即可，PC感知不到vlan的存在

  5. Trunk Link用交叉线，Access Link用直通线

RIP

路由信息协议(RIP) 是一种距离矢量协议，这表示它根据跳数来判断到达目标的最佳路由。

RIP防环机制

  • 定义最大度量值以防止计数至无穷大，规定最大跳数为16，凡是达到16跳为不可达路由

子接口实现跨VLAN访问

将GE0/0/1作为VLAN10用户的网关，GE0/0/2作为VLAN20用户的网关，可以实现两个VLAN的互访。

TRUNK

将 Switch1 和 Switch2 之间的链路设置为Trunk链路。

TRUNK的作用

  1. 传输VLAN
  2. 数据在trunk 链路上传输要打标签，数据在离开trunk链路时要脱标签
  3. 通过 trunk 的标签可以区分不同的流量。
  4. 注意：trunk 工作在数据链路层（只能识别 mac 地址），不能实现不同 vlan 之间的通信

TRUNK端口与access端口的区别

  • access端口一般只属于一个vlan，用于连接pc或router
  • trunk可以允许多vlan通过，用于交换机之间

ACL

指令 用法
配置access-list access-list [list number] [permit
将指定访问列表应用到相关接口，并指定ACL作用的方向 ip access-group {[access-list-number]&#124;[name]} [int&#124;out]
显示已设置的访问控制列表内容 show ip access-lists

ACL有什么用处

  1. 限制网络流量、提高网络性能；
  2. 提供网络安全访问；
  3. 可以在路由器端口决定哪种类型的通信流量被转发或被阻塞。

ACL配置原则

一个ACL可以由多条“deny/permit”语句组成，每一条语句描述一条规则，每条规则有一个Rule-ID。匹配顺序按照ACL的Rule-ID的顺序，从小到大进行匹配。

为每种协议、每个方向、每个接口配置一个ACL

ACL命令解释

  1. source address：主机ip地址
  2. 通配符掩码：“0”— 锁住。用来固定不能变的部分。“1”— 任意取值，用来表示放开的部分，不需要连续。
  3. host：0.0.0.0
  4. any：255.255.255.255
